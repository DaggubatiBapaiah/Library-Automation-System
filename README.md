
# Library Automation System

## Overview

Built a full-stack **Library Automation System** as part of a technical evaluation by **AB Tech Ville LLP** in January. Took complete ownership of the product—from system design to deployment—delivering a working solution under tight timelines.

## Problem

Library operations are typically manual, slow, and error-prone—especially when handling large inventories and tracking book locations.

## Solution

Designed and built a system that automates book identification, tracking, and management using computer vision and backend automation.

## What I Built

* End-to-end system for managing books, users, and transactions
* Computer vision pipeline (OCR + barcode + image recognition) for automated book identification (>90% accuracy)
* Sub-3-second book scanning workflow for faster operations
* Real-time hierarchical tracking system (Aisle → Rack → Row → Slot) for precise inventory location
* Scalable REST APIs enabling modular system communication
* Analytics dashboard for monitoring usage, performance, and inventory insights

## Tech Stack

* Python
* OpenCV
* Tesseract OCR
* FastAPI / Flask
* REST APIs
* Computer Vision & Image Processing

## Execution Mindset

* Prioritized speed + quality (built and shipped in 4–5 days)
* Focused on building a usable product, not just a prototype
* Structured the system for scalability and real-world application

## Outcome

The solution was well received, and I was offered an internship by **AB Tech Ville LLP** based on my performance.

## Why This Matters

* Demonstrates ability to build and ship quickly under pressure
* Combines backend engineering with applied computer vision
* Focused on solving real operational problems, not just technical tasks
